          Game Projects

          Here you find a collection of games I made. It is not a comprehensive list, but it contains the projects most important to my journey as a game developer.

          >> I use the Godot (game engine) and Aseprite (image editor for pixel art) as tools.

          Colony Defender 2

          Colony Defender 2 has the most impressive presentation of any game I made so far. Like it’s predecessor it is a shoot-em-up game, but instead of the Atari 2600 era, it is inspired by game for the SNE and Sega Genesis. The game is still in development, but the demo is available on Itch.io.

          Despite the simple premise, Colony Defender 2 is a complex game with many different components which must work together. Planning a file structure, building reusable templates and so on is in many ways just as important building the game itself.

          Play Code

          Slider Puzzle

          Slider Puzzle was another cute puzzle game, inspired by the NES and SNES era. Like Happy Blocks it was made for mobile use and with touch input in mind. This game was special however, as it was the first time, I made a game to order.

          Slider Puzzle can be played on Itch.io and is currently pending its clearance for the Google Playstore.

          Play Code

          Happy Blocks

          Happy Blocks is a cute little puzzle game, inspired by game for the NES and SNES gaming consoles. Your goal is to guide a little blue block through a maze and collect candy on your way.

          Happy Blocks was the first game I made specifically for smartphones and with touch controls in mind. It also was the first game I published on an official platform – the Google Playstore.

          Play Google Play

          Colony Defender

          Colony Defender is a simple 1-bit shoot-em-up in the style of Atari 2600 games. It was the first game I finished on my own, without following any tutorials.

          For me this was a very important step in game design. I stepped back from big, unrealistic ideas, and worked on something I was certain I could build and finish on my own. I learned how to research programming questions and overcome problems by disassembling them into smaller, more manageable steps.
